1. INTRODUCTION
================================================================================

Investidubh is a self-hosted OSINT platform for automated web intelligence 
collection and analysis.

Key Features:
  ‚Ä¢ Multi-source data collection (Web, RSS, SNS, GitHub, Certificate Transparency)
  ‚Ä¢ Automatic entity extraction (People, Organizations, Emails, Domains, IPs)
  ‚Ä¢ Relationship analysis and graph visualization
  ‚Ä¢ Temporal intelligence (data freshness evaluation)
  ‚Ä¢ Priority scoring (automatic identification of important entities)
  ‚Ä¢ Pattern detection (automatic anomaly detection)
  ‚Ä¢ PDF report generation

Built entirely with free, open-source tools with no external API dependencies.

================================================================================

„ÄêStep 1„ÄëClone the Repository

„ÄêStep 2„ÄëConfigure Environment Variables

    Open your browser and navigate to http://localhost:3000

5. BASIC USAGE
================================================================================

--------------------------------------------------------------------------------
5.1 Creating an Investigation
--------------------------------------------------------------------------------

  1. On the dashboard, click "New Investigation"
  2. Enter the target URL (e.g., https://example.com)
  3. Click "Create"
  4. Collection will start automatically

  Note: Status changes to "COMPLETED" when collection finishes

--------------------------------------------------------------------------------
5.2 Viewing Collected Data
--------------------------------------------------------------------------------

  1. Click on an investigation from the list
  2. "Artifacts" tab: Raw data (HTML, screenshots)
  3. "Intelligence" tab: Extracted entities list

--------------------------------------------------------------------------------
5.3 Graph Visualization
--------------------------------------------------------------------------------

  1. Click "Graph" in the navigation
  2. All investigation entities displayed as a network graph
  
  „ÄêGraph Controls„Äë
    ‚Ä¢ Drag: Pan the entire graph
    ‚Ä¢ Scroll: Zoom in/out
    ‚Ä¢ Click node: Show detail panel
    ‚Ä¢ Timeline slider: Filter by time

  „ÄêNode Colors and Shapes„Äë
    ‚Ä¢ Circle: People, organizations, domains, etc.
    ‚Ä¢ Square: Subdomains
    ‚Ä¢ Red border/glow: High priority entity
    ‚Ä¢ Orange border: Medium priority
    ‚Ä¢ Gray/faded: Old entity (ANCIENT)

--------------------------------------------------------------------------------
5.4 Entity Details
--------------------------------------------------------------------------------

  Click on a node to display the detail panel:
  
  ‚Ä¢ Basic Info: Type, value
  ‚Ä¢ Priority Score: 0-100 score with breakdown
  ‚Ä¢ Temporal Info: First seen, last seen, frequency
  ‚Ä¢ Freshness: FRESH (within 7 days) to ANCIENT (over 1 year)
  ‚Ä¢ Relationships: Links to related entities
  ‚Ä¢ Sentiment: Positive/negative score

--------------------------------------------------------------------------------
5.5 Search Function
--------------------------------------------------------------------------------

  1. Enter keywords in the search bar at the top
  2. Cross-search entities, artifacts, and investigations
  3. Click results to view details

6.4 Understanding Priority Scores
--------------------------------------------------------------------------------

  Scores range from 0-100 and are composed of 5 factors:

  | Factor                | Weight | Description                      |
  |-----------------------|--------|----------------------------------|
  | Degree Centrality     | 25%    | Number of connected edges        |
  | Frequency             | 20%    | Number of sightings              |
  | Cross-Investigation   | 25%    | Found in multiple investigations |
  | Negative Sentiment    | 15%    | Negative mentions                |
  | Freshness             | 15%    | Data recency                     |

  „ÄêScore Thresholds„Äë
    ‚Ä¢ 75-100: High priority (red glow)
    ‚Ä¢ 50-74: Medium priority (orange border)
    ‚Ä¢ 0-49: Low priority (normal display)

================================================================================

„ÄêProblem„ÄëServices won't start
  ‚Üí Check detailed logs with: docker-compose logs -f

8. FREQUENTLY ASKED QUESTIONS (FAQ)
================================================================================

Q: Is it free to use?
A: Yes. Investidubh is built entirely with free, open-source tools.
   There are no dependencies on external paid APIs.

Q: Where is data stored?
A: All data is stored in local Docker volumes.
   PostgreSQL (metadata), MinIO (raw files), Redis (queue).

Q: Can multiple users use it?
A: Yes. Data is isolated per user.
   Each user can only access their own investigations.

Q: Can it be used in production?
A: Internal network or VPN access is recommended.
   If exposing to public internet, configure HTTPS and reverse proxy.

Q: What kinds of sites can be investigated?
A: General websites, RSS feeds, SNS (Mastodon), GitHub repositories, etc.
   Please comply with robots.txt and terms of service.
